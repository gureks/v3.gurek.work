<title>Rewrite getSystemContext with Page Context and Redirect Heuristics</title>
<read_first>
- frontend/src/utils/chat.ts
- frontend/src/data/projects.ts
- frontend/src/App.tsx
</read_first>
<action>
Rewrite `frontend/src/utils/chat.ts` to:

1. Import `PROJECT_REGISTRY` from `../data/projects`.

2. Add a `getPageContext(pathname: string): string` function that returns a descriptive string:
   - `/` → `"User is on the home chat page — no templated content is displayed yet."`
   - `/projects` → `"User is viewing the Project Gallery with case study cards for: "` + project titles joined with commas
   - `/about` → `"User is viewing Gurek's resume, work experience, education, and professional background."`
   - `/playground` → `"User is browsing the interactive playground gallery."`
   - Paths starting with `/project/` → extract the slug, find it in `PROJECT_REGISTRY`, return `"User is viewing the deep-dive case study for: {title} — {shortDescription}"`
   - Fallback → `"User is on page: {pathname}"`

3. Rewrite `getSystemContext(pathname: string): string` to include:

   a. The existing ROLE & PERSONA section (keep as-is).
   
   b. A new `[CURRENT PAGE CONTEXT]` section that calls `getPageContext(pathname)`.
   
   c. An expanded `[AVAILABLE REDIRECTIONS]` section listing ALL routes:
      ```
      * / : Home chat page
      * /projects : Project Gallery showcasing built works
      * /about : Resume, work experience, and background
      * /playground : Interactive playground gallery
      * /project/et-epaper : Case study — ET ePaper
      * /project/growfast : Case study — GrowFast
      ... (all 6 project slugs from PROJECT_REGISTRY)
      ```
   
   d. A new `[REDIRECT DECISION RULES]` section with these exact heuristics:
      ```
      Analyze the user's query and choose ONE behavior:
      
      REDIRECT ONLY (redirect set, response is brief navigation context):
      - User explicitly asks to "show", "view", "browse", or "see" a section
      - User asks a vague question best answered by a whole page (e.g., "what have you built?")
      
      REDIRECT + ANSWER (both redirect AND substantive response):
      - User asks about a specific project by name AND wants to explore it → redirect to /project/<slug>, response summarizes the project
      - User asks about experience/resume AND wants details → redirect to /about, response highlights key points
      
      CHAT ONLY (redirect is null, answer inline):
      - User asks a specific technical question (e.g., "what tech stack did you use?")
      - User asks about a concept, role, or general career topic
      - User is already on the relevant page — NEVER redirect to ${pathname}
      
      IMPORTANT: Do NOT redirect just because a project name is mentioned. Only redirect when the user's INTENT is to navigate or explore, not when they ask a specific factual question.
      ```
   
   e. Keep the existing JSON output format section and constraints section.

4. Export both `getSystemContext` and `getPageContext`.
</action>
