<title>Create Toast UI Component</title>
<read_first>
- frontend/src/components/ui/Toast.tsx (to be created)
- frontend/src/store/useAppStore.ts
</read_first>
<action>
Create `frontend/src/components/ui/Toast.tsx`:

1. Import `useAppStore` and read `toast` and `dismissToast`.

2. Render a fixed-position notification at `top: 20px; right: 20px; z-index: 50`:
```tsx
import { useEffect } from 'react';
import { useAppStore } from '../../store/useAppStore';

export function Toast() {
  const { toast, dismissToast } = useAppStore();

  useEffect(() => {
    if (!toast) return;
    const timer = setTimeout(dismissToast, 4000);
    return () => clearTimeout(timer);
  }, [toast?.id, dismissToast]);

  if (!toast) return null;

  return (
    <div
      className="fixed top-5 right-5 z-50 max-w-sm animate-slide-in"
      style={{
        padding: '12px 16px',
        borderRadius: 'var(--radius-lg)',
        backgroundColor: 'var(--background-tooltip)',
        color: 'var(--foreground)',
        boxShadow: '0 4px 12px rgba(0, 0, 0, 0.15)',
        fontSize: '14px',
        lineHeight: '20px',
        animation: 'slideIn 300ms ease-out',
      }}
    >
      <p>{toast.message}</p>
    </div>
  );
}
```

3. Add the slideIn keyframe animation. Either add it inline or add to `index.css`:
```css
@keyframes slideIn {
  from { opacity: 0; transform: translateX(20px); }
  to { opacity: 1; transform: translateX(0); }
}
```

If the project's `index.css` already has a `@keyframes` section, add it there. Otherwise, use inline style with the animation property as shown above.
</action>

<action>
Modify `frontend/src/pages/ChatPage.tsx`:

1. Import `useAppStore` from `../store/useAppStore`.

2. Read navigation state for carried-over suggestions and toast:
```typescript
const locationState = location.state as { suggestions?: string[]; toast?: string } | null;
```

3. In a `useEffect` that runs when `location` changes:
   - If `locationState?.toast` is a non-empty string, call `useAppStore.getState().showToast(locationState.toast)`.
   - Clear the location state after consuming it to prevent re-showing on re-render:
   ```typescript
   useEffect(() => {
     if (locationState?.toast) {
       useAppStore.getState().showToast(locationState.toast);
       // Clear the state so it doesn't re-trigger
       window.history.replaceState({}, document.title);
     }
   }, [location.key]);
   ```

4. For suggestion carry-over:
   - If `locationState?.suggestions` exists and the current session has no messages (fresh arrival from redirect), use `locationState.suggestions` as the `activeSuggestions` instead of pulling from the last assistant message.
   ```typescript
   const carriedSuggestions = locationState?.suggestions;
   const lastAssistantMessage = [...messages].reverse().find(msg => msg.role === 'assistant');
   const activeSuggestions = (messages.length === 0 && carriedSuggestions?.length)
     ? carriedSuggestions
     : lastAssistantMessage?.suggestions;
   ```
</action>

- When `toast` is present in location state, `showToast` is called
- Location state is cleared after consuming toast (via `window.history.replaceState`)
- When session is empty and carried suggestions exist, they are used as `activeSuggestions`
- `ChatInput` receives carried suggestions when available
</acceptance_criteria>
