# Phase 5, Plan 02: Retry Logic, Graceful Degradation & Redirect+Answer Flow

<objective>
Add exponential backoff retry logic for failed JSON parsing, a "retrying…" indicator, graceful fallback to raw markdown, and the redirect+answer flow where the response renders before navigation occurs.
</objective>

<must_haves>
- Automatic retry on JSON parse failure (3 attempts with exponential backoff)
- Visual "retrying" indicator during retry attempts
- Graceful degradation: render raw response as markdown if all retries fail
- Redirect+answer: render message first, then navigate after brief delay
- Redirect-only: navigate immediately, skip adding empty response message
</must_haves>

Modify `frontend/src/store/useChatStore.ts`:

1. Add `retryAttempt: number | null` to the `ChatState` interface, initialized to `null`.

2. Create a helper function inside the file (not exported, module-level):

```typescript
const RETRY_DELAYS = [500, 1000, 2000];

function cleanAndParseJSON(raw: string): { redirect: string | null; response: string; suggestions: string[] } {
  // Strip markdown code fences
  let cleaned = raw.trim();
  cleaned = cleaned.replace(/^```(?:json)?\s*/i, '').replace(/```\s*$/g, '').trim();
  // Find the outermost JSON object
  const firstBrace = cleaned.indexOf('{');
  const lastBrace = cleaned.lastIndexOf('}');
  if (firstBrace !== -1 && lastBrace > firstBrace) {
    cleaned = cleaned.substring(firstBrace, lastBrace + 1);
  }
  return JSON.parse(cleaned);
}
```

3. Rewrite the `sendMessage` method's try block:

   a. After calling `sendQuery`, attempt `cleanAndParseJSON(replyText)`.
   
   b. If parsing fails, enter a retry loop:
      - For each attempt (up to 3): set `retryAttempt` state to the attempt number (1, 2, 3), wait `RETRY_DELAYS[i]` ms, re-call `sendQuery` with the same request, attempt parsing again.
      - If any attempt succeeds, clear `retryAttempt` to `null` and continue with the parsed response.
      - If all retries exhaust, clear `retryAttempt` to `null`, then apply graceful degradation (step 4).
   
   c. Full retry flow:
   ```typescript
   let parsedResponse;
   let rawText = replyText;
   try {
     parsedResponse = cleanAndParseJSON(rawText);
   } catch (parseErr) {
     // Retry loop
     let retrySuccess = false;
     for (let attempt = 0; attempt < RETRY_DELAYS.length; attempt++) {
       set({ retryAttempt: attempt + 1 });
       await new Promise(resolve => setTimeout(resolve, RETRY_DELAYS[attempt]));
       try {
         const retryResponse = await sendQuery({
           query: contentWithContext,
           mode: 'mix',
           conversation_history: history,
         });
         rawText = retryResponse.response.trim();
         parsedResponse = cleanAndParseJSON(rawText);
         retrySuccess = true;
         break;
       } catch (retryErr) {
         // Continue to next attempt
       }
     }
     set({ retryAttempt: null });
     if (!retrySuccess) {
       // Graceful degradation — render raw text as markdown
       if (rawText && rawText.length > 10 && /[a-zA-Z\s]/.test(rawText)) {
         parsedResponse = { redirect: null, response: rawText, suggestions: [] };
       } else {
         throw new Error('Response could not be parsed after retries');
       }
     }
   }
   ```

4. After parsing succeeds, implement the redirect+answer flow:
   
   ```typescript
   if (parsedResponse.redirect && navigate) {
     if (parsedResponse.response && parsedResponse.response.trim().length > 0) {
       // REDIRECT + ANSWER: render message first, then navigate
       const assistantMessage: ChatMessage = {
         id: crypto.randomUUID(),
         role: 'assistant',
         content: parsedResponse.response,
         suggestions: parsedResponse.suggestions || [],
         timestamp: Date.now(),
       };
       set((state) => ({
         sessions: {
           ...state.sessions,
           [pathname]: [...(state.sessions[pathname] || []), assistantMessage],
         },
         isLoading: false,
         retryAttempt: null,
       }));
       // Brief delay so user sees the response before redirect
       await new Promise(resolve => setTimeout(resolve, 1500));
       navigate(parsedResponse.redirect, {
         state: { suggestions: parsedResponse.suggestions },
       });
       return;
     } else {
       // REDIRECT ONLY: navigate immediately
       set({ isLoading: false, retryAttempt: null });
       navigate(parsedResponse.redirect, {
         state: {
           suggestions: parsedResponse.suggestions,
           toast: parsedResponse.response || null,
         },
       });
       return;
     }
   }
   ```

5. For non-redirect responses, keep existing logic but ensure `retryAttempt` is cleared to `null` on success.

6. In the catch block (error handler), also clear `retryAttempt: null`.
</action>
